## Important: Two-File Structure for AraBERT Fine-Tuning

During the primary `exp13_arabert_comparison.py` run (2026-03-16), the `finetune_arabert()` call
raised an exception on every fold (Trainer API / accelerate compatibility issue) and fell back
silently to the chance-level default (0.5). The summary JSON therefore records 0.5.

A dedicated re-run using the manual PyTorch training loop resolved this. Results are in
`arabert_finetuned_results.json` and confirmed by `../../exp13_finetuned_arabert.log`.

`visualize_exp13.py` loads `arabert_finetuned_results.json` and overrides the stale 0.5 values
when building paper figures. **Always use `arabert_finetuned_results.json` for the fine-tuned
AraBERT numbers**, not the `AraBERT_finetuned` field inside `exp13_summary.json`.
